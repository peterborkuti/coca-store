Beverages Inventory Application

I planned an app which stores how many beverages consumed from our office's inventory.

Plan I - Bar Code Reader

A bar code reader with a cheap web camera can be used. I tested it with ZBar but it worked, but with a fearly low probability. It can be improved with a special case made around the webcam and some LEDs for the appropriate lightning, but I rejected this idea because making a case, testing it and using can be a long running task

Plan II - Button controlled selector

A two-layered UI with buttons of Coca Cola products. First the buttons from here: products than the images of the real products.

I rejected it because it is not "geek" enough, however, I planned to code it.

Plan III - Voice Regognition System

User says a name of a product, or just some keywords, like "zero" (which can points to both Coca Cola Zero and Fanta Zero Orange), than user says "fanta" and boom, the product selected by the app. Than user says how many bottles (like 1 or 2 or 6) than "ok" or "cancel".
